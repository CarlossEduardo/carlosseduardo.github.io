Fazer um programa em Java, com interface gráfica e armazenamento dos dados em banco de dados(SGBD PostgreSQL), que permita fazer o controle de vendas de produtos em uma loja fictícia.

O programa de controle de vendas deverá extrair e exibir as seguintes informações:

1) Uma tela mostrando, em um período fornecido pelo usuário, a quantidade de vendas por dia, com os seguintes dados: a) Tabela de distribuição de frequência; b) Calcular e mostrar a média, moda, mediana, variância e desvio padrão; c) Visualizar um gráfico a partir da tabela. 2) Uma tela mostrando, em um período fornecido pelo usuário, o valor das vendas por dia, com os seguintes dados: a) Tabela de distribuição de frequência; b) Calcular e mostrar a média, moda, mediana, variância e desvio padrão; c) Visualizar um gráfico a partir da tabela. 3) Uma tela mostrando, em um período fornecido pelo usuário, os seguintes dados: a) Qual cliente compra mais vezes b) Qual cliente gasta mais. 4) Uma tela mostrando, em um período fornecido pelo usuário, uma tabela com a forma de pagamento e o cálculo da moda; 5) Fazer uma carta de controle do valor das vendas usando os limites a média e um desvio padrão e mostrar os valores fora dessa faixa através de um gráfico de dispersão.

Para esse banco de dados deverão ser produzidos os seguintes artefatos:

Modelo Entidade-Relacionamento (no formato de Chen); Modelo Entidade-Relacionamento (no formato pé-de-galinha); Script de criação do banco de dados para o SGBD PostgreSQL

Fazer o diagrama de Casos de Uso para o programa

a (com as descrições de cada Caso de Uso, conforme modelo definido em sala de aula); Para cada Caso de Uso, utilizando a metodologia de Pontos de Caso de Uso, mensurar o esforço para o desenvolvimento.

Curso

Análise de sistemas é a atividade que tem como finalidade realizar estudos de processos a fim de encontrar o melhor e mais racional caminho para que a informação possa ser processada. O analista de sistema estuda os diversos sistemas existentes entre hardwares (equipamento), softwares (programas) e o usuário final, seus comportamentos e aplicações, desenvolvendo a partir de então soluções que serão padronizadas e transcritas da forma que o computador possa executar.
